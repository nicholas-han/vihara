# 待定问题

设计中浮现、需由创始人拍板的决策。IV-vs-RV 策略细节正在讨论中(2026-06-21);**Q1** 与 **Q2**
标 **[阻塞纵切]**,因为它们在写任何代码前就定下"我们算什么";**Q7** 阻塞纵切的"跑起来"。每条都给出
建议以便快速推进。

## Q1 — IV 侧用哪个"隐含"数? **[阻塞纵切]**
拿已实现方差去对比单个 ATM 期权的隐含波动率,还是对比**无模型隐含方差**(方差互换公允执行价,
VIX 式,来自整条曲面)?
*建议:* 用 `asset_pricer` 的方差互换静态复制(v4)算无模型隐含方差——它才是与已实现*方差*同口径的
对手项,且用整条 smile 而非单一执行价。保留单点 ATM IV 作为更简单的兜底/对照。

## Q2 — 已实现方差的估计量族与采样 **[阻塞纵切]**
可配置采样频率(1s / 1min / 5min / 日频)是既定项。用哪个(些)估计量?如何处理微结构噪声、隔夜跳空、
年化?
*建议:* 有盘中数据时默认 **5 分钟已实现方差**(波动率 signature plot 的甜点);只有日频 OHLC 时用
**区间型估计量(Yang-Zhang)**。提供 **realized-kernel / two-scale** 估计量,以便对噪声鲁棒地使用更高
频率。按资产类年化(crypto 365×24 连续 vs 股票 252),与 `instrument_manager` ADR-21 对齐;隔夜跳空
处理做成显式配置开关。

## Q3 — 预测目标与期限
`forecaster` 到底预测什么、什么期限?
*建议:* **期权剩余期限内的前瞻已实现方差**(h 日前瞻 RV,匹配隐含期限),配 HAR 式多期限滞后特征
(1d / 5d / 22d)。如此预测值可直接与 Q1 的隐含数同口径比较。

## Q4 — 模型清单与填充顺序
在共享 `Forecaster` 协议之后,放哪些模型、什么顺序?
*建议:* **HAR-RV(Corsi)作基线**(RV 的标准模型),其后 EWMA / RiskMetrics,再 GARCH 族
(GARCH / EGARCH / GJR),再 ML(RF / GBM,基于 HAR 特征),再 DL(LSTM / TCN / Transformer)。一次填
一个;每个模型都在同一 walk-forward 切分上与 HAR 基线对比。

## Q5 — 信号构造(细节推迟)
VRP 信号怎么形成、怎么交易?
*建议:* 信号 = 隐含方差 − E[已实现方差](水平价差)和/或比值;做 z-score;阈值进出场。交易**delta
对冲后的方差头寸**(隐含 ≫ 预测时空方差 / 空跨式,隐含 ≪ 预测时多),用 `asset_pricer` 的 Greeks 以
现货/永续做 delta 对冲。等 IV、RV 两侧跑通后再细化。

## Q6 — 可交易标的与场所
哪个标的/场所(Deribit/Hyperliquid 的加密期权、股指期权……)?这决定行情源、执行适配器,以及方差互换
是否可直接交易还是要用期权复制。
*建议:* 等接 MarketData seam 时再定;策略逻辑保持与场所无关。推迟。

## Q7 — 纵切的数据源 **[阻塞跑起来]**
历史期权链 + 现货/盘中序列从哪来(vendor、平面文件、API)?
*建议:* 先用**静态历史文件**做可复现的首跑,藏在 `MarketData` seam(ADR-3)之后,将来换实时源即换
适配器。跑通前必须先有。
*注:* 加载器须产出在日历时间上连续(step-1)的决策行,否则 walk-forward 的 purge 需改到日历空间——
不然会在缺失日处 under-purge(见 `forecaster/validation/splitters.py`)。

## Q8 — IV 侧:Python 复制 vs `asset_pricer` pybind 绑定
纵切用纯 Python 离散复制(`DiscreteReplicationIV`)算无模型隐含方差。`asset_pricer` 已有 C++ 的连续
Carr-Madan / SVI 复制,但**还没有 Python 绑定**。
*建议:* 原始交易所链就用离散复制(交易所本就这么算,且无需 C++ 构建步骤);需要平滑曲面时,加一个
`asset_pricer` pybind 绑定去委托连续/SVI 版本。绑定现在建还是等 OKX 数据落地再建,待定。推迟。
